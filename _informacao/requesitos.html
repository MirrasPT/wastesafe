  1. ‌Introdução

    1. ‌Objectivo do Documento

      Descrever de forma completa, precisa e não ambígua todos os requisitos funcionais e não funcionais para a plataforma WasteSafer Tex. Este documento serve como base para o desenvolvimento, testes e aceitação do produto. É o contrato principal entre as partes interessadas (stakeholders) e a equipa de desenvolvimento.


    2. ‌Âmbito do Produto

      O sistema WasteSafer Tex será uma solução de e-commerce B2B/C2C (tipo OLX/Classificados) focada na Indústria Têxtil. O âmbito principal abrange:

      • Funcionalidades INCLUÍDAS:

        • Um marketplace focado na redução de desperdício têxtil, permitindo a transação de matérias-primas excedentárias, restos de coleções e peças com defeito.

        • Gestão de inventário simplificada (Anúncios com gestão de estado e expiração).

        • Pesquisa avançada com filtros (material, preço, localização, estado, qualidade).

        • Gestão de Perfis de Utilizador (Admin, Utilizador).

        • Monetização (Subscrições, Pagamento por Anúncio, Destaques, Verificação de Perfil).

        • Notificações (Push e Email) e Chat para negociação (Contacto Indireto).

        • Plataforma Web Responsiva e Aplicação Móvel (iOS/Android).

        • Um BackOffice de Administração (Gestão, Moderação, Relatórios).

        • Módulo de Favoritos e "Armazém Digital".

      • Funcionalidades EXCLUÍDAS:

        • Processamento de pagamentos de transações (Pagamento é externo).

        • Cálculo de comissões sobre vendas.

        • Logística e transporte (a negociação é feita entre as partes).

        • Leilões (modelo de preço fixo ou negociação direta).

        • Emissão de faturas fiscais (apenas recibos de subscrição/anúncio).

    4. ‌Restrições (General Constraints)‌

      • Tecnológicas (Backend): O Back-end DEVE ser desenvolvido em Laravel. A Base de Dados DEVE ser MySQL.

      • Tecnológicas (Frontend): A Aplicação Móvel (iOS/Android) e a Plataforma Web (Responsiva e BackOffice) DEVEM ser desenvolvidas usando Flutter.

      • NOTA PARA REVISÃO: Regulamentares/Normas: O sistema DEVE operar de acordo com a legislação e normas fiscais e de proteção de dados (RGPD) de Portugal. As regras a serem implementadas devem ser fornecidas pelo cliente.

      • Moeda: Todas as transações financeiras (Monetização) DEVEM ser processadas e apresentadas em Euros (EUR).

      • Limites de Tempo: O MVP (Minimum Viable Product) DEVE ser entregue até 31/12/2025.

      • Idiomas: O sistema DEVE suportar, no mínimo, Português (PT) e Inglês (EN).


    5. ‌Pressupostos e Dependências (Assumptions and Dependencies)‌

      • Pressuposto (Assumption): A integração com Gateways de Pagamento é viável em Portugal para cobrança de serviços da plataforma.

        • Stripe Billing: DEVE ser usado para pagamentos recorrentes (Subscrições).

        • Stripe Checkout / SDK Flutter: DEVE ser usado para pagamentos únicos (Anúncios, Destaques, Serviços).

      • Dependência (Dependency): O desenvolvimento do Módulo de Monetização depende do fornecimento das API Keys dos Gateways de Pagamento (Stripe, PayPal, MBWay).

      3. ‌Módulo de Pesquisa, Matching e Favoritos‌


        • RF_201: O sistema DEVE fornecer uma barra de pesquisa de texto livre.

        • RF_202: O sistema DEVE fornecer filtros avançados, incluindo (no mínimo): Tipo, Localização, Preço (Min/Max), Estado de Preservação, Qualidade (RF_103), Certificações.

        • RF_203: O sistema DEVE fornecer um matching básico (V1.0) que sugere produtos relevantes a compradores com base nas suas pesquisas anteriores.

        • RF_204: O sistema DEVE permitir ao Utilizador "Guardar Pesquisas" para receber alertas de novos produtos.

        • RF_205 (Gestão de Favoritos): O sistema DEVE permitir a um Utilizador marcar (e desmarcar) um anúncio como "Favorito".

        • RF_206 (Lista de Favoritos): O sistema DEVE fornecer uma secção no perfil do Utilizador ("Meus Favoritos") para ver os anúncios guardados.

        • RF_207 (Algoritmo de Ranking da Pesquisa): Para além dos filtros (RF_202), o sistema DEVE ordenar os resultados da pesquisa com base nos seguintes fatores de relevância:

          • RF_207.1 (Slots de Destaque): Os primeiros X resultados (ex: Top 3) DEVEM ser reservados exclusivamente para Anúncios com Destaque Pago (RF_1001).

            • Regra de Seleção de Destaques: Dentro do grupo de Anúncios Destacados, o sistema DEVE ordenar pela relevância da expressão de pesquisa (texto/filtros), tal como nos resultados orgânicos. Se a pesquisa for vazia, a ordem dos destaques é aleatória ou cronológica.

          • RF_207.2 (Ranking Orgânico): Após os slots de destaque, a ordem dos anúncios restantes DEVE ser influenciada positivamente pelos seguintes atributos do Vendedor e do Produto:

            • a. Relevância Textual: Correspondência com a expressão de pesquisa.

            • b. Rating Médio: (RF_902) - Classificação mais alta tem prioridade.

            • c. Quantidade de Vendas Concluídas: (Histórico) - Maior volume de vendas tem prioridade.

            • d. Quantidade de Produtos Ativos: (Anúncios à venda) - Maior quantidade tem prioridade.

            • e. Completude do Perfil: (Ex: Foto de perfil, Descrição) - Perfis 100% completos têm prioridade.

        • RF_1102: A Disputa DEVE permitir o upload de provas (fotos, descrição) e bloquear o

          Chat associado para revisão do Admin.

        • RF_1103 (Resolução): O Admin (Suporte Técnico, RF_805) DEVE poder rever a disputa e emitir uma recomendação (Nota: Como o pagamento é externo, o Admin não pode reverter fundos, apenas mediar ou suspender o utilizador infrator).

  1. ‌Sugestões de Trabalho Futuro‌

    Esta secção apresenta funcionalidades e melhorias identificadas como valiosas para a evolução da plataforma WasteSafer Tex, mas que estão excluídas do orçamento e âmbito da Versão 1.0 (MVP).

    1. Integração de Inteligência Artificial (IA):

      • Matching Preditivo: Implementação de algoritmos de Machine Learning para sugerir produtos com base no histórico de navegação e compras do utilizador, indo além dos filtros básicos.

      • Previsão de Dead Stock: Análise de padrões de upload para alertar as empresas sobre o risco de acumulação de stock antes que este se torne obsoleto.

    2. Internacionalização (i18n):

      • Expansão da plataforma para outros mercados europeus (Espanha, França), incluindo suporte multi-moeda e multi-idioma avançado.

    3. Logística Integrada:

      • Parceria com transportadoras para oferecer cotações de envio em tempo real e geração de etiquetas de envio diretamente na plataforma.

    4. Pagamentos In-App (Escrow):

      • Evolução para um modelo de marketplace gerido, onde a WasteSafer Tex processa o pagamento da transação e retém o valor em custódia até à confirmação da entrega, aumentando a segurança.

    5. Integração ERP:

      • Desenvolvimento de APIs ou plugins para integração direta com softwares de gestão de stocks (ERP) das empresas têxteis (ex: SAP, PHC, Primavera), automatizando o upload de produtos.
